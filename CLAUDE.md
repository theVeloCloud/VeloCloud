## Project Overview

**VeloCloud v3** is a modular Minecraft cloud system for orchestrating server instances (Paper, Velocity, etc.) across Local, Docker, and Kubernetes runtimes. It is an early-stage prototype, not production-ready.

Current version: `3.0.2` (configured in `build.gradle.kts` and `gradle/libs.versions.toml`).

## Build & Run Commands

All modules require **Java 21** (JVM toolchain enforced via Gradle).

```bash
# Build the distributable launcher fat JAR (includes all submodules)
./gradlew :launcher:jar

# Build a specific module
./gradlew :agent:jar

# Run all tests
./gradlew test

# Run tests for a specific module
./gradlew :platforms:test

# Run a single test
./gradlew :platforms:test --tests "de.snenjih.velocloud.platforms.PlatformTest.loadPool"

# Build Docker image (native architecture)
./gradlew :launcher:dockerBuild
# Or directly:
docker build -t velocloud:development -f docker/Dockerfile .
```

The launcher output is `launcher/build/libs/velocloud-launcher.jar`.

## Module Architecture

The project is a Gradle multi-module build (`settings.gradle.kts`):

```
launcher   → thin Java bootstrapper (downloads deps, starts agent via classloader)
agent      → main cloud orchestration process (Kotlin)
common     → shared utilities (Kotlin: filesystem, JSON, i18n, version, math, OS)
platforms  → platform/version definitions for Minecraft server software (Kotlin)
updater    → GitHub release version checker (Kotlin)
```

**Dependency graph:** `launcher` bundles `agent + common + platforms + updater`. The `agent` depends on all others plus two external artifacts: `de.snenjih.velocloud:proto` and `de.snenjih.velocloud:shared` (gRPC stubs and shared interfaces, fetched from Maven Central snapshots).

### Launcher (`launcher/`)

Pure Java module. Downloads runtime dependencies listed in `src/main/resources/dependencies.blob` (generated by `buildDependencies` task), then starts the agent via a custom classloader. The `dependencies.blob` is regenerated each build from the resolved `:agent` runtime classpath.

### Agent (`agent/`)

The core. Entry point: `AgentBoot.kt` → `main()` → initializes logging → instantiates the `Agent` singleton object.

`Agent` extends `VelocloudShared` (from the external `shared` artifact) and wires together:
- **`Runtime`** — selected automatically at startup: Kubernetes > Docker > Local (fallback). Each runtime implements group storage, service storage, factory, expender, template storage, and config holder.
- **`GrpcServerEndpoint`** — starts a gRPC server (default port `8932`) exposing services for events, groups, services, players, platform info, templates, and cloud information.
- **`ModuleProvider`** — loads external module JARs at runtime.
- **`SecurityProvider`** — filters server group access.
- **`EventService`** — internal cloud event bus.
- **`DetectorFactoryThread` / `OnlineStateDetector`** — background thread that monitors service online states.

Boot sequence: `Runtime.initialize()` → `Agent.boot()` → reads `AgentConfig` → loads modules → starts gRPC → calls `Runtime.prepareBoot()` → starts `serviceStatsThread()` → `Runtime.boot()`.

Config file (`AgentConfig`) defaults: port `8932`, auto-update `true`, max 4 concurrent server starts, max 4 caching processes, max 75% CPU before blocking starts.

### Runtime Implementations

| Runtime | Detection | Notes |
|---|---|---|
| `KubernetesRuntime` | Kubernetes env available | Uses fabric8 client |
| `DockerRuntime` | Docker socket accessible | Uses docker-java; mounts host socket |
| `LocalRuntime` | Always (fallback) | Process-based, includes JLine terminal |

The `LocalRuntime` also provides the interactive terminal (JLine 3). Docker runtime requires `DOCKER_HOST=unix:///var/run/docker.sock`.

### Platforms (`platforms/`)

Defines available Minecraft server platforms (Paper, Velocity, etc.) and their download tasks. Platform definitions are loaded from `metadata/platforms/` and task steps from `metadata/tasks/` (copied into resources at build time by the `copyTasks`/`copyPlatforms` tasks). `PlatformPool` is the singleton registry.

### Common (`common/`)

Pure utility library. Key packages: `filesystem`, `json` (Gson-based), `network`, `language` (i18n), `version`, `math`, `os`, `image`.

### Updater (`updater/`)

Checks GitHub releases against the current version to determine if an update is available. Used by `Agent` to warn or auto-update on startup.

## i18n

All agent-facing messages go through `i18n` (an `I18nVelocloudAgent` instance). Translation files live in `agent/src/main/resources/i18n/velocloud-agent_<locale>.properties`. Translations are managed on Crowdin.

## Publishing

SNAPSHOT builds publish to `central.sonatype.com/repository/maven-snapshots`. Release builds (version not ending in `-SNAPSHOT`) publish to Sonatype staging. Credentials are `ossrhUsername`/`ossrhPassword` environment variables. CI runs `./gradlew :common:publish` on every push to `master`.
